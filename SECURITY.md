# Security Policy

## Supported versions

Only the latest released version of dbt-semantic-mcp receives security updates.
Older versions are unsupported.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security
vulnerabilities.

Use GitHub's private vulnerability reporting feature instead:
**[Report a vulnerability](https://github.com/in-loop/dbt-semantic-mcp/security/advisories/new)**.

Alternatively, email the maintainer directly: ctyoungb@gmail.com. PGP
key on request.

We'll acknowledge receipt within **3 business days** and aim to publish
a fix or mitigation within **30 days** of the initial report. If the
issue is critical and exploited in the wild, we'll move faster.

## Disclosure policy

We follow coordinated disclosure: a fix lands in a release first; the
public advisory follows within 7 days of the release. Reporters are
credited unless they request anonymity.

## Scope

In-scope:
- Code in this repository.
- Released artifacts published from this repository (binaries, container
  images, Python wheels, npm packages, etc.).

Out of scope:
- Third-party dependencies — please report those upstream.
- Issues in deployment environments not controlled by this project.

## Verifying release artifacts

All releases are signed with **cosign keyless** via GitHub Actions OIDC
(Decision 12). Verify a release artifact:

```bash
cosign verify-blob \
    --certificate-identity-regexp '^https://github.com/in-loop/dbt-semantic-mcp/' \
    --certificate-oidc-issuer https://token.actions.githubusercontent.com \
    --bundle <artifact>.bundle \
    <artifact>
```

Each release also publishes:
- An **SBOM** (SPDX + CycloneDX) generated by syft.
- An **SLSA provenance** statement.

Both are signed with the same cosign keyless flow.
